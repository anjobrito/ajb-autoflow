# AJB AutoFlow — Documentação Comercial

## Visão geral

O **AJB AutoFlow** é um sistema web em desenvolvimento pela **AJBSYSTEMS** para gestão operacional de negócios automotivos.

A proposta é atender oficinas mecânicas, auto centers, lava-rápidos, estética automotiva e empresas que precisam organizar clientes, veículos, funcionários, ordens de serviço e histórico de atendimentos.

O projeto está atualmente em fase de MVP, com funcionalidades essenciais sendo construídas e validadas antes da evolução para uma plataforma SaaS completa.

---

## Problema que o sistema busca resolver

Muitos pequenos e médios negócios automotivos ainda dependem de controles manuais, planilhas, mensagens de WhatsApp, cadernos ou sistemas genéricos que não acompanham bem o fluxo real da operação.

Isso gera problemas como:

- dificuldade para encontrar o histórico de um veículo;
- perda de informações de clientes;
- falta de controle das ordens de serviço;
- retrabalho no atendimento;
- baixa rastreabilidade dos serviços realizados;
- dificuldade para acompanhar produtividade;
- falta de visão financeira integrada;
- ausência de padronização operacional.

O AJB AutoFlow nasce para organizar esse fluxo de forma simples, visual e progressiva.

## Funcionalidades atuais do MVP

A versão atual contempla a base inicial do sistema.

### Cadastro de clientes

Permite registrar informações básicas dos clientes para consulta e uso nos fluxos operacionais.

### Cadastro de veículos

Permite associar veículos aos clientes, criando uma base organizada para histórico e ordens de serviço.

### Cadastro de funcionários

Permite iniciar uma base de funcionários/responsáveis operacionais, preparando o sistema para evolução de comissões e produtividade.

### Ordens de serviço

Permite iniciar o controle de atendimentos e serviços executados.

### Histórico operacional

Permite consultar informações relacionadas aos atendimentos registrados.

### Interface padronizada

O sistema possui uma identidade visual e estrutural em evolução, buscando manter consistência entre as telas.

## Modelo comercial planejado

A proposta futura é evoluir o AJB AutoFlow para um modelo SaaS.

SaaS significa que o cliente acessa o sistema pela internet, mediante uma assinatura ou plano de uso.

Uma estrutura comercial futura pode considerar:

```txt
Plano Inicial
- Cadastro de clientes
- Cadastro de veículos
- Ordens de serviço básicas
- Histórico simples

Plano Profissional
- Recursos do plano inicial
- Funcionários
- Comissões
- Financeiro básico
- Relatórios

Plano Empresa
- Recursos avançados
- Multiusuários
- Multiempresa ou multiunidade
- Integrações
- Suporte prioritário
```

Essa estrutura ainda é apenas uma visão comercial planejada, não uma oferta definitiva.
